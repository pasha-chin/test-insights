## Что было сделано

### 1. Lazy loading компонентов
- **Проблема:** весь JS грузился сразу, блокируя рендер
- **Решение:** `React.lazy()` + `Suspense` для графиков и дашборда
- **Эффект:** TBT 10ms → 0ms, FCP −1.1s

### 2. Code splitting (разделение чанков)
- **Проблема:** один большой bundle
- **Решение:** вынесли `vendor` и `charts` в отдельные чанки через `vite.config`
- **Эффект:** LCP −1.9s, Performance +38 пунктов

### 3. Локальные шрифты
- **Проблема:** запрос к Google Fonts блокировал рендер
- **Решение:** шрифты скачаны и подключены локально через `@font-face`
- **Эффект:** убран внешний blocking request, FCP улучшился

### 4. Фиксированная высота элементов
- **Проблема:** CLS 0.313 — элементы прыгали при загрузке
- **Решение:** явные `height`/`min-height` для карточек и skeleton-заглушек
- **Эффект:** CLS 0.313 → 0.017

### 5. Бинарный поиск стартового оффсета + ранний выход из пагинации (backend)
- **Проблема:** VK API не фильтрует по дате — без оптимизации нужно перебирать все посты группы с начала
- **Решение:** для групп > 500 постов — бинарный поиск стартового оффсета (`findStartOffset`), затем линейный проход только нужного диапазона с ранним выходом при `post.date < fromDate`
- **Эффект:** для группы с 10 000 постов вместо ~100 запросов к API — ~7 (log₂) на поиск оффсета + минимум на сам период
